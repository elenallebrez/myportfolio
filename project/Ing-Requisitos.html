Proyecto de Requisitos

Documentación y análisis de requisitos para un sistema software

Ingeniería de Requisitos Casos de uso Especificación Modelado UML

Ver documentacion

Resumen

El problema: antes de programar un sistema, es clave entender qué necesitan los usuarios y qué restricciones existen. La solución: elaboré dos documentos de requisitos detallando actores, objetivos, casos de uso, reglas de negocio y requisitos funcionales y no funcionales. El resultado: una especificación clara que serviría como base sólida para un futuro desarrollo de software.

Detalles técnicos

  • Enfoque: Ingeniería de requisitos, modelado UML, reglas de negocio
  • Arquitectura: Documento de requisitos, casos de uso, modelo conceptual, restricciones
  • Mi rol: Análisis y redacción de requisitos
  • Tiempo: 3 meses, 2025
  • Estado: Documento finalizado

Cómo lo construí

  1. Identificación de actores y objetivos del sistema
  2. Definición de requisitos funcionales, no funcionales y reglas de negocio
  3. Diseño de diagramas UML y consolidación en un documento formal

Resultados & aprendizajes

  • Resultados: Documento de requisitos completo, con casos de uso, reglas de negocio y modelos
  • Aprendizajes: Importancia de analizar antes de programar, cómo estructurar requisitos claros y evitar ambigüedad
