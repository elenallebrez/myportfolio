Java Term Project

Academic project in Java to process and analyze real baseball data, developed during my first year of university.

Java Collections Streams OOP

View on GitHub

The challenge was to design a system capable of working with a real dataset (baseball Statcast data) while applying object-oriented programming concepts. My approach started by exploring the dataset itself: understanding what kind of insights could be extracted and how to leverage the data effectively. I then implemented domain classes, object factories, and data structures (maps and collections) to represent and query the data. The result was a structured Java program that performed meaningful analyses (e.g. comparing players or plays) using collections and streams.

Technical Details

  • Stack: Java, Collections, Streams
  • Architecture: Domain classes + object factory + collection-based containers
  • My role: Full development — dataset exploration, data modeling, and implementation of operations
  • Duration: 4 months, 2023
  • Status: Completed and delivered

How I built it

  1. Explored the Statcast dataset to identify interesting metrics and insights.
  2. Designed domain classes to represent pitches, games, and players.
  3. Implemented collections and dictionaries (maps) to organize data efficiently.
  4. Applied streams to query, filter, and analyze data in a functional style.
  5. Structured the solution with factories and modular design for maintainability.
