Food Delivery System (CRUD + REST API)

Food ordering platform connecting restaurants and customers

Node.js Express MariaDB React Native REST API

View on GitHub

Restaurants need a digital platform to manage orders and reach more customers. That’s why we designed a complete system with backend and mobile apps for both customers and restaurant owners. The result: a functional MVP inspired by Glovo/UberEats, including order management, product catalogues, restaurant listings, and user accounts.

Technical Details

  • Stack: Node.js, Express, MariaDB, React Native
  • Architecture: Client-server with REST API; separate mobile apps for customers and owners
  • My Role: Backend development for customer orders and frontend implementation for the customer app
  • Timeline: 4 months, 2025
  • Status: Academic project (functional MVP)
How it was built
  1. We were given the initial web platform schema and requirements in class (business rules, functional and non-functional requirements).
  2. I implemented the backend in Node.js and Express, connecting it to MariaDB for data persistence.
  3. We built a REST API to handle all CRUD operations (restaurants, products, and orders) following the rules (e.g., free shipping for orders > €10, one restaurant per order, etc.).
  4. I also contributed to the React Native app for customers, integrating API endpoints and ensuring smooth order creation and tracking.
  5. Finally, we documented everything (requirements, use cases, and technical architecture) and tested scalability and security using libraries like Sequelize and Helmet.
